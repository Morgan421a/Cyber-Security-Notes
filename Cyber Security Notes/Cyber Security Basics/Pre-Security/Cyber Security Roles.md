
**Security Analyst -** 

Analyse cyber security within a company, 

form reports regarding network safety, 

document security issues and steps taken in response, 

Builds security plans following new attack tools and trends as well as the required steps amongst teams to ensure data security is maintained.


**Security Engineer -**

Uses threat and vulnerability data to build and implement security solutions

Tests and screens security measures across software

Monitors networks and reports to update systems and mitigate vulnies

Spot and implement systems needed for optimal security


**Incident Responder -**

Protect a company's data, rep and financial standing from cyber attacks

High pressure

Use incident response metrics MTTD, MTTA, MTTR

Develops and uses a feasible incident response plan

Ensuring strong security practices are maintained and supporting incident response measures

Post-incident reporting and prep for future attacks, learning and adapting from incidents


**Digital Forensics Examiner -**

Collect and analyse evidence to solve crimes (Law enforcement)

Analyse incidents such as policy violations (Corporate)

Gather and analyse digital evidence while observing legal procedures to find answers relating to the case

Document findings and report on the case


**Malware Analyst -**

Analyse suspicious programs and reporting on what they do

Also called a reverse-engineer

Converts compiled programs from machine language (often assembly) into readable code typically in a low level language like C. 

Requires a strong programming background especially in low level langs such as a ASM or C

Entails finding out about what a malicious program does, how to spot it and report it.

Uses reverse-engineering to do static analysis of malicious programs

Carries out dynamic analysis of malware samples by observing them in a controlled environment

Documents and reports all of their findings


**Penetration Tester -**

Often referred to as a Pen tester sometimes called ethical hackers

Tests the security of a company's systems and software

Look for vulnies and flaws in system security via systemised hacking

Exploit discovered vulnies to evaluate risk in each instance allowing a company to shore up their security or plan to defend these weak points in the event of a real cyber attack


**Red Teamer -**

More targeted version of a pen tester

Test a company's detection and response capabilities

Often carried out by an external team
