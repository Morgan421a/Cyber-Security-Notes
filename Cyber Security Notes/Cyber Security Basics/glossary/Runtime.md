1. The last stage in the program lifecycle when code is actively running on a device, during which
	- Computer allocates memory, loads binaries, and executes instructions
	- Some errors may only occur in this phase called **runtime errors** i.e. dividing by 0, missing files, etc

2. Refers to the software infrastructure (virtual layer between the code and the OS) needed to execute a program, providing essential services such as:
	- Memory management - allocating and freeing memory (e.g. garbage collection)
	- Libraries - Pre-written code for common tasks (file I/O, network requests)
	- Security - Enforcing safety rules (e.g. preventing a program from accessing restricted memory)
